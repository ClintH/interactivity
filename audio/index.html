audio

Each of these examples is has two parts - a sketch that works with analysed data, and another sketch which does the analysis. The latter are called 'remotes'. You can, for example run a remote on your mobile phone (thus using its microphone), and further processing and using the data on your laptop.

Remember that Glitch projects are accessible from https://PROJECTNAME.glitch.me

  • playground: experiment with processing settings
  • playground-meyda: experiment with audio features from the Meydar library
Sketches for extending. They are listed in ascending complexity.
  • threshold: detecting if audio hits a certain level
  • beats: detecting tempo from audio peaks
  • damper: demos the Damper helper class for (very crude) background noise cancellation
  • thing: A thing which changes form with current and historic audio

  • remote: a general purpose 'remote' for sending FFT-processed audio
  • client: a skeleton client for remote data
